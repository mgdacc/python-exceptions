El Problema: Errores

Cuando algo sale mal, Python detiene el programa y lanza una excepción.

resultado = 10 / 0

# El programa se detiene aquí con un:
# ZeroDivisionError: division by zero
2 / 9

La Solución: try...except

El bloque `try` te permite probar código riesgoso, y `except` lo captura si falla.

try:
    resultado = 10 / 0
except:
    print("¡Error! No se puede dividir por cero.")

# El programa continúa y no se detiene.
3 / 9

Capturar Errores Específicos

Es mejor capturar excepciones específicas para saber qué error ocurrió.

try:
    numero = int("hola")
except ZeroDivisionError:
    print("Error de división.")
except ValueError:
    print("Error de valor: no es un número.")

Acceder al Mensaje de Error

Puedes capturar el objeto de la excepción usando `as` para ver el mensaje original.

try:
    resultado = 10 / 0
except ZeroDivisionError as e:
    print(f"Ocurrió un error: {e}")

La Cláusula else

El bloque `else` se ejecuta solo si el bloque `try` termina sin errores.

try:
    resultado = 10 / 2
except ZeroDivisionError:
    print("Error de división.")
else:
    print(f"El resultado es: {resultado}")

La Cláusula finally

Este bloque se ejecuta SIEMPRE, haya ocurrido un error o no. Ideal para limpiar recursos.

try:
    # f = open("archivo.txt")
    # ... código ...
except:
    print("Error.")
finally:
    print("Este bloque siempre se ejecuta.")
    # f.close()

Lanzar Excepciones: raise

Puedes lanzar tus propias excepciones para forzar un error cuando algo no es válido.

def set_edad(edad):
    if edad < 0:
        raise ValueError("La edad no puede ser negativa.")
    print(f"Edad guardada: {edad}")

set_edad(-5) # Esto lanzará el error.
